# タスク実行時の自動 Worktree 分離 (Auto Worktree Isolation)

## Summary
`kage` のタスク実行時、ユーザーが現在作業しているリポジトリ（またはブランチ）を直接操作するのではなく、タスク専用の一時的な Git Worktree を自動生成してその中で実行する仕組みの提案。

## Problem
現在 `kage` は `working_dir` などで指定されたディレクトリ（多くの場合、現在の作業リポジトリ）上で直接コマンドや AI を実行します。
ユーザーが日中作業して未コミットの変更（dirty state）を残したまま夜間に `kage` を走らせた場合、以下の問題が発生します。
1. **変更の混入**: AI の変更と人間の未コミット変更が混ざってしまい、翌朝の切り分けが困難になる。
2. **破壊のリスク**: AI が誤って人間の作業中ファイルを上書きしたり、破損させたりするリスクがある。
3. **複数タスクの競合**: 複数の `kage` タスクが並列で実行された場合、同じファイルツリーを同時に操作して競合や不整合を起こす。

## Proposal
タスクのフロントマターに `isolation: worktree` を設定できるようにします（またはデフォルトの挙動とする）。

```yaml
---
name: Overnight Refactoring
cron: "0 2 * * *"
mode: autostop
isolation: worktree
---
```

この設定が有効な場合、`kage` は実行開始時に以下のプロセスを自動で行います。
1. **Worktree の作成**: リポジトリ内の一時ディレクトリ（例: `.kage/worktrees/{task_slug}`）に、現在の `main` などを起点とした一時的な Worktree を作成します。
2. **コンテキストの移動**: AI のファイル読み書きやシェルコマンド実行のワーキングディレクトリをその Worktree 内に自動で切り替え、作業を隔離された環境内に閉じ込めます。
3. **実行と破棄**: タスク完了時（「Auto PR Creation」機能と連動してPR作成後など）、不要になった Worktree を安全に `git worktree remove` で破棄します。

## Inspiration
- **easy-worktree**: 並行作業を Worktree で分離する運用フロー。AI エージェントにも人間と同じように「タスクごとの独立した作業空間」を与える思想。
- **CI/CD Pipelines (GitHub Actions)**: ジョブごとに毎回クリーンなチェックアウト環境を用意する仕組み。

## Expected Daily Benefit
- **精神的安全性 (Peace of Mind)**: ユーザーは日中の作業を中途半端な状態で放置したまま PC をスリープさせても、夜間の `kage` タスクが自分の作業ツリーを汚染する心配が一切なくなります。
- **並行実行の安定化**: 複数の `kage` タスクが完全に独立したファイルシステム上で動くため、ファイルロックやコンフリクトを気にせず並列スケジュールを組めます。

## Scope and Non-Goals
- **Scope**: `git worktree` コマンドの自動発行、`kage` 実行プロセスのカレントディレクトリの透過的な切り替え、後始末のライフサイクル管理。
- **Non-Goals**: Git 管理されていないプロジェクトでの仮想的なファイルシステム分離（Docker サンドボックスがその役割を担うため、本提案はあくまで Git リポジトリベースの分離に特化）。

## Risks or Open Questions
- **.kage ディレクトリの同期**: 一時 Worktree 内で生成された `.kage/memory/` や `kage.db` などのステート変更が、本体ツリー側と正しく同期される（または本体側のパスを絶対パスで参照する）工夫が必要です。
- **依存関係のインストールコスト**: Worktree を分けた場合、`node_modules` や `.venv` などの依存関係が空になり、タスク実行のたびにインストールが走る（遅くなる）可能性があります。キャッシュの共有機構の検討が必要です。

## Suggested Rollout
1. まずはタスクの `working_dir` を使って手動で作成済みの Worktree を指定するベストプラクティスをドキュメント化する。
2. 次に、フロントマターで `isolation: worktree` をサポートし、実行時のフックとして Worktree 生成と破棄を自動化する。
