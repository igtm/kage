# 複数プロジェクト横断の Global Unified Dashboard

## Summary

マシン上で稼働している複数の `kage` プロジェクト（リポジトリ）のタスク実行状況やログを、単一の Web UI または CLI コマンドで一覧できる「Global Dashboard」機能の提案。

## Problem

現在の `kage` はプロジェクト（ディレクトリ）ごとに `.kage/` を初期化し、個別のスコープで自律エージェントを稼働させる設計になっています。
この設計はリポジトリへのポータビリティが高く優れていますが、ユーザーが複数のプロジェクト（例: `frontend-app`, `backend-api`, `personal-blog` など）でそれぞれ夜間タスクを設定し始めると、朝に結果を確認するために各ディレクトリへ移動して `kage ui` や `kage task list` を叩き直す必要が生じます。
「自身のマシンで現在いくつのエージェントが、どのプロジェクトで動いているのか（または失敗しているのか）」を俯瞰する手段がなく、複数プロジェクト運用時の state visibility が低い状態です。

## Proposal

複数の `kage` プロジェクトを横断して管理・監視できる Global なビューを提供します。

1. **プロジェクトの自動登録**:
   各ディレクトリで `kage init`、`kage cron install`、または `kage run` が実行された際、その絶対パスをグローバル設定（例: `~/.kage/projects.json` や `~/.kage/kage.db` の projects テーブル）に自動登録します。
2. **Global CLI の追加**:
   カレントディレクトリに依存せず、全プロジェクトのタスク状態を一覧する `kage task list --global` コマンドを追加します。
3. **Global UI Dashboard**:
   `kage ui --global`（または特定のポートで常にグローバル用UIを立ち上げるモード）を提供し、左サイドバー等でプロジェクトを切り替えつつ、全タスクの実行履歴や次回のスケジュール、直近のエラー状況を1つの画面で確認できるようにします。

## Inspiration

- **Docker Desktop / Portainer**: 個別のコンテナ（プロジェクト）を横断して一元管理できるダッシュボード。
- **PM2 / pm2-gui**: 複数ディレクトリで起動している Node.js プロセスのステータスを単一のリストで監視する体験。
- **Airflow**: 複数の DAG (プロジェクト相当) を横断してステータスや実行履歴を一覧できる UI。

## Expected Daily Benefit

朝起きた際、1つのブラウザタブ（または1回のコマンド）を開くだけで、全プロジェクトの夜間バッチ（コード監査、テスト実行、PoC作成など）が成功したか失敗したかを瞬時に把握できます。これにより、各リポジトリを巡回する認知負荷と手間がゼロになり、エラーが起きているプロジェクトにだけ素早くフォーカスできるようになります。

## Scope and Non-Goals

- **Scope**: 登録された全 `kage` プロジェクトの SQLite DB や設定を読み取り、読み取り専用（または限定的な操作）の統合ビューを提供すること。
- **Non-Goals**: 1つのプロジェクト（ディレクトリ）から他のプロジェクトのタスクを直接実行・編集するような複雑な依存関係やリモート操作機能はスコープ外とします。あくまで「監視と一覧（Visibility）」に特化します。

## Risks or Open Questions

- **パフォーマンス**: 多数のプロジェクトが登録された場合、各ディレクトリの `kage.db` を都度ロードして UI に描画すると、I/O 負荷や表示の遅延が発生しないか。
- **プロジェクトの削除検知**: ユーザーがディレクトリ自体を `rm -rf` した場合、グローバルレジストリに「Ghost (Not Found)」として残ってしまう問題。UI 上で検知してレジストリからパージする仕組みが必要です。

## Suggested Rollout

1. **Phase 1**: `kage init` や `kage cron install` 時の `~/.kage/projects.json` へのパス自動登録（および手動での `kage project add/remove` コマンドの実装）。
2. **Phase 2**: CLI 側での `kage task list --global` の実装（各ディレクトリのステータスを集計して表示）。
3. **Phase 3**: `kage ui --global` における複数プロジェクト横断ビューの実装。
