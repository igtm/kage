# タスク作成とデバッグを容易にする Dry-Run モード (`--dry-run`)

## Summary

新規に作成したタスクが意図通りに解釈され、正しいプロンプトとメモリが構築されているかを、実際に AI API を呼び出したり副作用を起こすことなく確認できる `--dry-run` モードの提案。

## Problem

`kage` はバックグラウンドで自律稼働することを前提としているため、ユーザーが新しいタスク (`.kage/tasks/*.md`) を記述した際のデバッグ（Task Authoring）が困難です。
「現在のメモリがどのようにプロンプトに注入されているか」「`working_dir` が正しく解決されているか」「どのツールが AI に公開されているか」を確認するには、実際に `kage run` を実行して API コストを消費し、AI の挙動を見るしかありません。
特にシェルコマンドを実行できる権限を持たせている場合、意図しない破壊的変更（ファイルの誤削除など）を起こすリスクがあり、タスクのテスト実行への心理的ハードルが高くなっています。

## Proposal

CLI コマンドに `kage run <task_name> --dry-run` オプションを追加します。

1. **実行シミュレーション**:
   通常の実行フローと同じように、設定ファイル、フロントマターの解析、依存関係 (`depends_on`) のチェック、および `.kage/memory/` からの履歴読み込みを行います。
2. **API 呼び出しのインターセプト**:
   AI エンジン（Gemini, Claude 等）へリクエストを送信する直前で処理を停止します。
3. **ペイロードのダンプ出力**:
   AI に送信される予定だった「システムプロンプト」「ユーザープロンプト（注入されたメモリを含む）」「利用可能なツール（Functions）の一覧」を、コンソール上に JSON または Markdown 形式で分かりやすく出力して終了します。シェルコマンドモードの場合は、実行予定のコマンド文字列を出力します。

## Inspiration

- **GitHub Actions**: `act` ツールなどによるローカルでの事前動作確認体験。
- **Kubernetes / Helm**: `--dry-run=client` によるマニフェスト適用前の検証。
- **Terraform / Terraform Plan**: インフラ変更前の差分・実行計画の可視化。

## Expected Daily Benefit

新しい夜間タスクを思いついた際、「とりあえず書いてみて Dry-Run でプロンプトの組み立てを確認する」という安全な試行錯誤ループが回せるようになります。API コストの無駄遣いや、間違ったスクリプト実行によるローカル環境の破壊を防ぐ（Safe-guard）ことができ、Task Authoring の DX と安心感が劇的に向上します。

## Scope and Non-Goals

- **Scope**: タスク実行の準備段階（プロンプト構築、環境変数の解決、設定値の評価）をシミュレートし、AI API 呼び出しやシェル実行の直前でダンプ出力して安全に終了する機能。
- **Non-Goals**: AI の応答自体をモック（Mock）して後続の処理（ファイルの書き込みや Connector への通知）まで通しでシミュレーションすることは複雑すぎるためスコープ外とします。あくまで「入力（リクエスト）の検証」に特化します。

## Risks or Open Questions

- **状態の汚染**: Dry-Run 実行時に `kage.db` の「最終実行時刻」や「ステータス」を誤って更新してしまわないよう、データベースへの書き込み処理も適切にスキップ（またはトランザクションのロールバック）する設計が必要です。
- **Connector の扱い**: Connector による通知処理（開始通知など）も、Dry-Run 時にはコンソール出力へのルーティングに切り替える（ミュートする）必要があります。

## Suggested Rollout

1. **Phase 1**: コアロジックに `dry_run` フラグを引き回し、AI クライアントやシェルエグゼキューターの直前でリクエスト内容を stdout に dump して終了する基本実装。
2. **Phase 2**: `kage.db` への状態書き込みのバイパスと、Connector 通知のミュート処理の徹底。
3. **Phase 3**: `kage ui` 上での「テスト実行 (Dry Run)」ボタンの実装と、UI 上でのプロンプトプレビュー機能の提供。
